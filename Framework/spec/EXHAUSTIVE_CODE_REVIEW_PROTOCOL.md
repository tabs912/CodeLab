# Exhaustive Code Review Protocol

Use only when explicitly requested.

Required phases:

1. Repository and file inventory
2. Function inventory and call graph
3. Static error review
4. Execution-path review
5. Dependency audit
6. Library and public API review
7. Performance review
8. Data-integrity and deletion review
9. Trigger, concurrency, and locking review
10. Error-handling and logging review
11. Security and authorization review
12. Maintainability review
13. Duplicate, dead, and orphan-code audit
14. Testability review
15. Severity classification

Required deliverables:

- Executive Summary
- Findings Register
- Function and Dependency Inventory
- Public API Report
- Orphan and Duplicate Report
- Performance Report
- Data Integrity Risk Report
- Trigger and Concurrency Report
- Remediation Plan
- Test Plan

Do not modify code during the audit.
