# Code Production Standard

Production updates must:

- Preserve approved business logic
- Increment the version
- Replace complete affected functions whenever practical
- Update callers and dependencies
- Remove obsolete implementations only when proven safe
- Avoid unrelated changes
- Return complete replacement files
- Include release notes
- Include a test plan
- Identify intentionally unchanged known issues

Do not leave placeholders, TODOs, stubs, or incomplete code.
